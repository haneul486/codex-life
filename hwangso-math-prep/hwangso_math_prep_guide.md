## 현재 작업 순서

- 이 폴더의 작업은 아래 순서로 진행한다.
  1. 메인 지침과 메인 HTML에 시험 준비 기준과 일정표를 먼저 반영한다.
  2. 현재 수준 평가표를 만든다.
  3. 평가 결과를 기준으로 약점 유형과 HBI 수준을 기록한다.
  4. 시험일 전까지 주간 보정 계획과 실전 연습 자료를 추가한다.
- 현재 수준 평가는 단순 체크리스트가 아니라 인쇄 가능한 시험지 형태를 기본으로 한다.
- 시험지, 모의평가, 오답 점검표는 황소 폴더 안에 각각 독립된 하위 폴더로 만든다.
- 진단형 세트와 별도로, 실제 시험 감각을 보는 `실전판` 세트도 별도 하위 폴더로 관리한다.
- 각 시험 하위 폴더에는 가능하면 아래 파일을 둔다.
  - 인쇄용 HTML 파일
  - 필요하면 PDF 파일
  - 시험 목적과 범위를 적은 짧은 안내 파일
- 새로 만드는 시험 자료는 항상 메인 `hwangso_math_prep.html`에서 링크로 열 수 있어야 한다.
- 시험지 HTML의 각 문항에는 손으로 충분히 풀이할 수 있는 넉넉한 풀이 공간을 둔다.
- 기본적으로 풀이칸은 짧은 메모용이 아니라 계산 과정과 중간 정리를 쓸 수 있는 높이로 만든다.
- 사용자가 실제로 푼 시험지 사진이나 PDF를 올리면 같은 시험 폴더 안에 보관하고, 결과 리뷰 HTML에서 문항별 판정과 함께 바로 다시 열 수 있게 연결한다.
- 결과 리뷰 HTML에는 가능하면 제출 이미지 위에 `맞음`, `틀림`, `혼합` 같은 시각 표시를 얹어 한눈에 채점 상태를 볼 수 있게 만든다.
- 사진이나 스캔본으로 채점할 때는 한 번 읽고 바로 확정하지 말고, 문항별 답안을 최소 두 번 이상 다시 읽어 `1차 판독`과 `리체크 판독`을 거친 뒤 최종 판정을 적는다.
- 손글씨가 흐리거나 문항 요구가 여러 개인 경우에는 `확정`, `재확인 필요`를 나눠 적고, 확실하지 않은 내용을 억지로 단정하지 않는다.

## HBI 수준 기준

- `HBI`는 `Hwangso Benchmark Index`의 약자다.
- 목적은 황소수학 입학시험 수준을 학교 시험, 일반 심화 문제, 경시형 문제와 구분해서 일관되게 표현하는 것이다.
- 점수 방식은 6개 축을 각각 `0~4점`으로 평가하고 총점 `0~24점`으로 본다.

### 평가 축

- `A. 개념 결합도`
  - 0: 한 개념만 바로 적용
  - 1: 같은 단원 안의 두 개념 연결
  - 2: 다른 소단원 개념을 함께 사용
  - 3: 둘 이상의 개념을 순서 있게 결합
  - 4: 숨은 조건을 찾아 개념 구조를 재구성해야 함
- `B. 추론 단계 수`
  - 0: 한 번 계산하면 끝남
  - 1: 2단계 풀이
  - 2: 3단계 풀이
  - 3: 경우 나누기 또는 역추적이 필요
  - 4: 여러 조건을 동시에 점검하며 전략을 바꿔야 함
- `C. 표현 전환도`
  - 0: 식만 세우면 됨
  - 1: 문장을 식으로 바꾸면 됨
  - 2: 표, 그림, 수직선, 배열 같은 보조 표현이 유리함
  - 3: 표현을 한 번 바꿔야 핵심이 보임
  - 4: 적절한 표현을 스스로 선택하지 못하면 풀이가 막힘
- `D. 계산 정확도 부담`
  - 0: 단순 계산
  - 1: 연산 실수만 조심하면 됨
  - 2: 중간값 관리가 필요
  - 3: 계산보다 순서 통제가 중요
  - 4: 계산 실수 유도 요소가 강하고 검산이 필수
- `E. 비정형성`
  - 0: 학교 익힘책형
  - 1: 숫자만 바뀐 응용형
  - 2: 낯선 질문 방식이 섞임
  - 3: 처음 보는 구조지만 배운 개념으로 풀 수 있음
  - 4: 문제의 의도를 해석하는 데 시간이 많이 듦
- `F. 시간 압박`
  - 0: 시간 여유 충분
  - 1: 보통 수준
  - 2: 2분대 풀이 리듬이 필요
  - 3: 막히면 바로 건너뛰는 운영이 필요
  - 4: 빠른 선별, 재도전, 검산 운영까지 요구

### 총점 구간 해석

- `HBI-1 (0~5점)`: 학교 기본 문제
- `HBI-2 (6~10점)`: 학교 심화 또는 일반 학원 응용
- `HBI-3 (11~15점)`: 사고력 입문 또는 경시 입문
- `HBI-4 (16~19점)`: 황소 입학시험 표준 구간
- `HBI-5 (20~24점)`: 황소 상위권 또는 고배점 압박 구간

### 실제 적용 규칙

- 황소 입학시험 대비 문제를 만들 때 기본 목표 구간은 `HBI-4`로 본다.
- 실전 세트는 아래 비율을 기본값으로 삼는다.
  - `HBI-3` 30%
  - `HBI-4` 50%
  - `HBI-5` 20%
- 특정 문항이 `초4-1 범위` 안에 있어도 `HBI-2` 수준이면 황소 대비용으로는 약한 문제로 분류한다.
- 반대로 선행 개념이 없어도 추론, 비정형성, 시간 압박이 강하면 `HBI-4` 또는 `HBI-5`로 분류할 수 있다.
- 앞으로 문제 분석, 오답 정리, 모의문항 생성, 주간 계획은 모두 `단원명 + HBI 레벨 + 약점 태그` 형식으로 기록한다.

- `rangeAnalysis`: 초등학교 4학년 1학기 수학 단원, 핵심 개념, 자주 틀릴 포인트, 예상 문제 유형을 담는다.

## 수정할 때 우선 확인할 부분

- 날짜는 `YYYY-MM-DD` 형식으로 유지한다.
- 확인된 정보와 아직 확인이 필요한 정보를 분리해서 적는다.
- 입학시험 관련 일정과 범위는 이 지침 파일을 기준으로 먼저 반영한 뒤 HTML 데이터에도 같은 내용을 맞춘다.
- 시험범위는 단원명만 적지 말고 핵심 개념, 자주 나오는 유형, 취약 포인트까지 연결해서 적는다.
- 황소수학 수준 분석은 막연한 인상보다 `HBI` 6개 축으로 나눠 기록한다.
- 공개 자료가 부족하거나 시험지가 비공개인 부분은 추정이라고 표시하고, 공식 근거와 비공식 근거를 섞어 쓰지 않는다.
- 일정표는 `진단 → 약점 보정 → 실전 운영` 흐름이 보이도록 유지한다.
- 인쇄용 시험지는 A4 인쇄를 고려한 HTML을 우선 만들고, 필요하면 PDF를 추가한다.
- 인쇄용 시험지의 풀이칸은 실제 손풀이를 기준으로 넉넉하게 확보하고, 문항당 풀이 공간이 너무 작지 않은지 먼저 확인한다.
- 종합 문제나 HBI-4 이상 문제는 일반 문항보다 더 큰 풀이칸이 필요한지 함께 검토한다.
- 메인 HTML에서 새 시험 자료 링크가 빠지지 않았는지 항상 확인한다.
- 결과 리뷰 문서는 빈 양식으로 두지 말고, 실제 채점이 끝나면 `총점`, `문항별 판정`, `약점 태그`, `다음 학습 우선순위`까지 구체적으로 채워 둔다.
- 사진 채점 결과를 기록할 때는 최종 반영 전에 `총점`, `문항별 정오`, `사진별 맞음·틀림 표시`가 서로 일치하는지 마지막으로 한 번 더 확인한다.
- 화면 문구는 학부모나 학생이 바로 판단할 수 있는 정보 중심으로 짧게 유지한다.
- 정적 마크업을 직접 늘리기보다 데이터 배열이나 객체를 먼저 수정하고 렌더링 결과를 맞춘다.

## 인쇄 및 공유

- 필요하면 상담 또는 준비 회의용으로 인쇄할 수 있게 단색 위주의 인쇄 스타일을 유지한다.
- 외부 라이브러리 없이 단일 HTML 파일로 열리게 유지한다.
- 인쇄용 HTML에는 `@page`를 사용해 기본 페이지 크기를 `A4` 기준으로 잡는다.
- 인쇄 모드에서는 화면용 `max-width`, 큰 바깥 여백, 카드형 그림자 효과 때문에 브라우저가 과하게 축소하지 않도록 인쇄 전용 폭과 여백을 다시 지정한다.
- 인쇄 모드에서는 `main`, `sheet`, 본문 래퍼의 `max-width`를 해제하고 종이 폭에 맞게 다시 배치한다.
- 화면에서는 3열, 4열로 보이는 카드나 정보 박스도 인쇄 시에는 1열 또는 2열로 재배치해서 페이지 폭을 넘지 않게 한다.
- 표와 그리드가 인쇄 폭을 넘겨 페이지의 일부만 작게 축소되어 찍히지 않도록 인쇄 전용 레이아웃을 별도로 확인한다.
- 시험지, 채점표, 판정표를 새로 만들 때는 브라우저 인쇄 미리보기에서 실제 출력 비율이 정상인지 확인하는 것을 우선 점검 항목으로 둔다.
