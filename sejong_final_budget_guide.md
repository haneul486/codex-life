# 인테리어 예산표 작성/수정 지침

이 문서는 [sejong_final_budget.html](C:/Users/haneu/OneDrive/문서/New project/sejong_final_budget.html) 파일을 유지보수할 때 참고하는 상세 작업 지침이다.

## 1. 문서 목적

이 HTML 문서는 다음 목적을 가진다.

- 세종인테리어 공사비를 확정값으로 관리한다.
- 가전, 가구, 수납, 생활비 등 구매 필요항목을 하나씩 채워가며 총 예산을 계산한다.
- 화면에서는 상세 입력표를 볼 수 있어야 한다.
- 인쇄/PDF 저장 시에는 상세 입력표를 숨기고 요약 중심 문서로 출력할 수 있어야 한다.

즉, 이 문서는 비교 견적서가 아니라 `최종 예산 관리표`다.

## 2. 파일 구성 원칙

현재 문서는 크게 네 영역으로 구성된다.

1. 상단 헤더
2. 요약 카드 4개
3. 금액 입력된 구매 목록 요약표
4. 공사비 표
5. 구매 필요항목 상세 입력표

이 구조는 유지하는 것을 기본 원칙으로 한다.

특히 다음 규칙을 유지한다.

- 상단 헤더 제목은 `인테리어 예산표`
- 공사비는 `세종인테리어` 기준
- 요약 카드에는 `확정 공사비`, `구매 예정 합계`, `0원 체크 항목`, `총 예상 필요금`만 둔다
- 상세 입력표는 실제 편집용이다
- 상단 요약표는 상세 입력표를 자동 반영하는 읽기 전용 결과물이다

## 3. 금액 단위 규칙

모든 입력값은 `만원` 단위다.

예:

- `4600` 입력 = `46,000,000원`
- `165` 입력 = `1,650,000원`
- `10` 입력 = `100,000원`

수정 시 반드시 이 단위를 유지한다.

HTML 내부 계산은 다음 방식으로 작동한다.

- 입력값 숫자 x `10000`
- 계산 결과는 원 단위로 표시
- 요약 카드에서는 `만원` 또는 `억/만원` 형식으로 축약 표시

## 4. 현재 핵심 데이터 의미

### 공사비

- `constructionCost` 입력값
- 현재 기본값은 `4600`
- 이 값은 확정 공사비로 사용된다

### 구매 필요항목

- `purchaseTable` 안의 각 `input type="number"`가 예산 항목이다
- 각 행은 `분류`, `항목`, `메모`, `예상 금액`, `상태`로 구성된다

### 자동 계산 항목

다음 값들은 수동 입력하지 않는다.

- `확정 공사비`
- `구매 예정 합계`
- `0원 체크 항목`
- `총 예상 필요금`
- 상단 `금액 입력된 구매 목록`
- 하단 `구매 예정 합계`

이 값들은 모두 스크립트가 자동 계산한다.

## 5. 항목 추가 규칙

새 항목을 추가할 때는 반드시 `purchaseTable`의 `tbody` 안에 한 줄을 추가한다.

기본 형식:

```html
<tr>
  <td>분류</td>
  <td>항목명</td>
  <td>메모</td>
  <td class="money"><span class="amount-input"><input type="number" value="0">만원</span></td>
  <td><span class="status">0원</span></td>
</tr>
```

### 분류 작성 규칙

분류는 상단 요약표의 카테고리 그룹핑 기준이 된다.

가능한 기존 분류를 재사용한다.

- `가전`
- `가구`
- `수납`
- `생활`
- `주방`
- `관리`
- `임시비`
- `예비`

새 분류를 만들 수는 있지만, 상단 요약표가 그 분류명 그대로 묶기 때문에 너무 잘게 쪼개지지 않게 한다.

### 상태 작성 규칙

- 금액이 확정되었으면 `status fixed`
- 아직 검토 중이면 `status todo`
- 0원 체크 항목이면 기본 `status`

예:

```html
<span class="status fixed">입력</span>
<span class="status todo">검토</span>
<span class="status">0원</span>
```

## 6. 항목 수정 규칙

기존 항목 수정 시 다음만 바꾸는 것이 원칙이다.

- 항목명
- 메모
- `input`의 `value`
- 상태 텍스트

스크립트 함수 이름, `id`, 요약 카드 구조는 가능하면 건드리지 않는다.

## 7. 항목 삭제 규칙

항목을 삭제할 때는 `purchaseTable`의 해당 `<tr>` 전체를 제거한다.

단, 삭제 전에 아래를 확인한다.

- 상단 요약표에서 자동 반영되는지
- 합계가 원하는 방향으로 줄어드는지
- 같은 의미의 항목이 다른 이름으로 중복 존재하지 않는지

## 8. 상단 요약표 동작 원리

상단 `금액 입력된 구매 목록` 표는 `0원이 아닌 항목만` 보여준다.

동작 방식:

- 상세 입력표 전체를 읽는다
- `amount > 0`인 항목만 필터링한다
- 분류별로 묶는다
- 각 분류 소계를 계산한다
- 항목은 2열 형태로 나열한다

즉, 어떤 항목을 상단에 보이게 하려면 `0보다 큰 값`을 넣으면 된다.

보이지 않게 하려면 `0`으로 두면 된다.

## 9. 인쇄 동작 규칙

현재 인쇄는 일반 `window.print()`가 아니라 `handlePrint()` 함수를 통해 실행된다.

이 함수는 다음 순서로 동작한다.

1. `body`에 `printing-mode` 클래스 추가
2. 인쇄 창 실행

CSS에서는 `body.printing-mode .detail-section`을 숨긴다.

즉:

- 화면에서는 상세표가 보인다
- 인쇄 버튼을 눌렀을 때만 상세표가 숨겨진다
- 새로고침하면 다시 원래 화면으로 돌아온다

이 동작은 유지한다.

### 인쇄 관련 수정 시 주의

다음을 깨뜨리면 안 된다.

- `#printButton`
- `handlePrint()`
- `body.printing-mode .detail-section`
- `@media print`
- `@page size: A4`

## 10. A4 PDF 출력 기준

이 문서는 A4 PDF 저장을 염두에 두고 작성되었다.

인쇄 CSS의 목표:

- A4 세로 기준
- 여백 10mm
- 카드와 표가 너무 크지 않게 축소
- 상세 입력표는 인쇄에서 제외
- 상단 헤더, 요약 카드, 상단 요약표, 공사비 표만 남김

만약 인쇄 레이아웃을 다시 바꿔야 한다면 다음 순서로 조정한다.

1. `@page`의 크기와 margin 확인
2. `@media print`에서 `.page` 너비 확인
3. `.grid`의 열 수와 gap 조정
4. `th`, `td`, `input` 글자 크기와 패딩 조정
5. 상세 입력표가 계속 숨겨지는지 확인

## 11. 색상/디자인 유지 원칙

현재 문서는 강한 초록 계열을 버리고 차콜/블루그레이 계열로 정리되어 있다.

유지 원칙:

- 헤더: 차콜 + 블루그레이 그라데이션
- 버튼: 블루그레이
- 카드 포인트색: 블루그레이, 슬레이트, 샌드, 클레이
- 표와 배경: 회색 계열

다음은 피한다.

- 형광색에 가까운 초록
- 채도 높은 보라
- 지나치게 선명한 파랑
- 과한 그림자

## 12. 스크립트 수정 규칙

핵심 함수는 다음 네 개다.

- `getConstructionTotal()`
- `getPurchaseRows()`
- `getPurchaseTotal()`
- `updatePricedItemsSummary()`
- `updateTotals()`

### 함수 역할

`getConstructionTotal()`
- 공사비 입력값을 원 단위로 변환

`getPurchaseRows()`
- 상세 입력표의 각 항목을 읽어서 배열로 반환

`getPurchaseTotal()`
- 구매 항목들의 총합 계산

`updatePricedItemsSummary()`
- 0원 초과 항목만 분류별로 묶어 상단 요약표를 렌더링

`updateTotals()`
- 전체 카드와 합계 텍스트를 갱신

이 함수 이름과 흐름은 가능하면 유지한다.

## 13. 한글 깨짐 방지 규칙

이 파일은 UTF-8로 저장되어야 한다.

주의:

- 다른 인코딩(CP949 등)으로 저장하면 한글이 깨질 수 있다
- 파일을 대량 치환하거나 외부 편집기에서 저장할 때 인코딩을 꼭 확인한다
- 제목, 헤더, 버튼, 표 헤더, 스크립트 문자열 안 한글도 모두 UTF-8이어야 한다

깨짐이 생겼을 때 증상 예:

- `????`
- `媛`
- `留뚯썝`

이런 문자가 보이면 인코딩 깨짐으로 보고 파일 전체를 점검한다.

## 14. 값 변경 시 검산 포인트

값을 바꾼 뒤 최소한 아래를 확인한다.

1. 구매 예정 합계가 맞는지
2. 총 예상 필요금이 `공사비 + 구매 예정 합계`와 같은지
3. 0원 항목 개수가 맞는지
4. 상단 요약표에 0원 아닌 항목만 보이는지
5. 인쇄 버튼 클릭 시 상세표가 빠지는지
