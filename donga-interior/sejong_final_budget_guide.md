## 0. 수정 기준 원칙

이 문서를 수정할 때는 **항상 사용자가 마지막으로 직접 수정해둔 실제 파일 상태를 기준으로 작업합니다.**

즉, 아래 원칙을 따릅니다.

- 먼저 현재 [sejong_final_budget.html](C:/sky/project/codex-life/donga-interior/sejong_final_budget.html) 파일 내용을 확인합니다.
- 이전 대화에서 기억한 값이나 예전 버전을 기준으로 덮어쓰지 않습니다.
- 사용자가 HTML 안에서 직접 바꿔둔 금액, 문구, 배치, 상태값이 있으면 그 상태를 유지한 채 필요한 부분만 수정합니다.
- 한글이 깨졌거나 구조가 무너진 경우에도, 가능하면 현재 파일의 최신 사용자 수정분을 보존하면서 복구합니다.

간단히 말해, **“대화에서 말했던 예전 상태”보다 “지금 파일에 실제로 들어 있는 상태”를 우선 기준으로 삼습니다.**

## 1. 문서 목적

이 예산표는 아래 항목을 한 화면에서 같이 관리하기 위한 문서입니다.

- 공사비 총액과 지급 상태
- 이미 구매한 항목
- 앞으로 구매할 항목
- 전체 예상 필요금

## 2. 현재 화면 구조

현재 문서는 아래 순서로 구성됩니다.

1. 상단 제목
2. 요약 카드 4개
3. 공사비 지급 현황
4. 구매 상태 요약
5. 구매 필요항목

`비용 부담 요약` 섹션은 현재 사용하지 않으므로 제거된 상태입니다.

## 3. 상단 요약 카드

상단에는 4개의 카드가 있습니다.

- `확정 공사비`
- `기구매 합계`
- `미구매 예정 합계`
- `총 예상 필요금`

계산 기준:

- 확정 공사비: 공사비 지급 현황 표의 금액 합계
- 기구매 합계: 구매 상태가 `구매완료`인 항목 합계
- 미구매 예정 합계: 구매 상태가 `구매예정`인 항목 합계
- 총 예상 필요금: `확정 공사비 + 기구매 합계 + 미구매 예정 합계`

## 4. 공사비 지급 현황

공사비는 아래 단계로 나누어 관리합니다.

- 계약금
- 중도금 1
- 중도금 2
- 잔금

표 컬럼:

- 구분
- 지급일
- 금액(만원)
- 지급 경로
- 상태

상태 값:

- `지급완료`
- `예정`

현재 기본 입력값:

- 계약금: `300만원`, `2026-04-25`, `국민 632902-01-074940 이하종`, `지급완료`
- 중도금 1: 현재 HTML 입력값 기준
- 중도금 2: 현재 HTML 입력값 기준
- 잔금: 현재 HTML 입력값 기준

공사비 숫자를 바꾸면 상단 카드도 자동 반영됩니다.

## 5. 구매 상태 요약

이 표는 `구매 필요항목`의 내용을 자동으로 모아 보여주는 요약 표입니다.

표시 규칙:

- 예상 금액이 `0`보다 큰 항목만 표시
- 카테고리별로 묶어서 표시
- 구매완료 항목은 `구매완료` 라벨 표시
- 구매완료 항목은 실결제 금액이 있으면 실결제 금액 기준으로 표시
- 실결제 금액이 없으면 예상 금액 기준으로 표시

즉, 이 표는 직접 수정하는 표가 아니라 아래 상세표를 바꾸면 같이 갱신되는 자동 요약 영역입니다.

## 6. 구매 필요항목

실제 예산 관리를 하는 핵심 표입니다.

컬럼:

- `분류`
- `항목`
- `메모`
- `예상(만원)`
- `실결제(원)`
- `상태`
- `구매일`
- `결제 경로`

상태 값:

- `구매예정`
- `구매완료`
- `보류`

금액 규칙:

- `예상(만원)`은 만원 단위
  - 예: `165` = `1,650,000원`
- `실결제(원)`은 원 단위 그대로 입력
  - 예: `234740`

## 7. 날짜 입력칸 동작

`구매일`, `지급일`의 빈 날짜칸은 기본 `mm/dd/yyyy` 같은 안내 문구가 화면에 보이지 않도록 처리되어 있습니다.

동작 방식:

- 값이 비어 있으면 빈 칸처럼 보임
- 날짜를 고르면 실제 날짜가 표시됨

## 8. 현재 반영된 주요 항목

현재 문서에 기본 입력된 주요 항목 예시는 아래와 같습니다.

- 냉장고 `165`
- 김치냉장고 `200`
- 워시타워 `192`
- 로봇청소기 `50`
- 게이트맨 도어락 `25`
- 에어컨 `185`
- 붙박이장 `200`
- TV장 `100`
- 퀸침대 `200`
- 슈퍼싱글침대 `150`
- 협탁 2개 `40`
- 화장대 `50`
- 식탁 / 의자 세트 `200`
- 아일랜드장 `30`
- 낮은 수납장 `70`
- 발코니 수납 선반 3개 `30`
- 한달숙소비 / 복비 등 `150`
- 매트리스 커버 2개 `10`
- 생활살림 `50`

구매완료로 반영된 기본 항목:

- 게이트맨 도어락
  - 실결제 `234,740원`
  - 구매일 `2026-04-27`
  - 결제 경로 `쿠팡 결제`

## 9. 인쇄 규칙

문서의 `인쇄` 버튼을 누르면:

1. `구매 필요항목` 상세표가 숨겨짐
2. 브라우저 인쇄 창이 열림
3. 인쇄가 끝나면 화면 상태가 돌아옴

즉, 인쇄본에는 너무 긴 하단 상세표를 넣지 않고 상단 요약 중심으로 출력합니다.

## 10. 수정 원칙

수정할 때는 아래 순서를 권장합니다.

1. 공사비 변동이 있으면 `공사비 지급 현황`부터 수정
2. 구매한 항목은 상태를 `구매완료`로 변경
3. 실결제 금액, 구매일, 결제 경로를 함께 입력
4. 아직 살 예정이면 `구매예정` 유지
5. 아직 판단 전이면 `보류` 유지

## 11. 수정 후 체크리스트

- 상단 4개 요약 카드 숫자가 맞는지
- 구매 상태 요약에 금액 있는 항목만 나오는지
- 구매완료 항목에 `구매완료` 라벨이 붙는지
- 빈 날짜칸에 기본 날짜 문구가 안 보이는지
- 인쇄 버튼을 누르면 상세표가 숨겨지는지
- 한글이 깨지지 않는지

## 12. 문자 인코딩 규칙

이 프로젝트의 HTML, Markdown, 텍스트 파일은 모두 `UTF-8` 기준으로 유지합니다.

특히 아래 항목은 항상 확인합니다.

- HTML 안의 한글 제목, 본문, 버튼 문구
- JavaScript 문자열
- JavaScript 주석
- Markdown 문서 본문

한글이 깨진 상태를 발견하면 부분 수정으로 덧대지 말고, UTF-8 기준으로 다시 저장하고 확인합니다.
