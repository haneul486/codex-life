## 3. 현재 예산표 구조

예산표 상단에는 네 개의 요약 카드가 있습니다.

### 3-1. 확정 공사비

- 공사 계약 기준 총 공사비
- 현재 기준 금액: `4,600만원`
- 아래 `공사비 지급 현황` 표의 금액 합계와 연결됩니다

### 3-2. 기구매 합계

- 이미 구매 완료한 항목들의 합계
- 구매 상태가 `구매완료`인 항목만 포함됩니다
- 실결제 금액이 입력되어 있으면 실결제 금액을 우선 사용합니다

### 3-3. 미구매 예정 합계

- 아직 구매하지 않았지만 예산을 잡아둔 항목들의 합계
- 구매 상태가 `구매예정`인 항목만 포함됩니다
- 예상 금액 기준으로 합산합니다

### 3-4. 총 예상 필요금

- 계산식:
  - `확정 공사비 + 기구매 합계 + 미구매 예정 합계`

## 4. 공사비 지급 현황 표

공사비는 총액만 보는 것이 아니라 지급 단계별로 나눠서 관리합니다.

현재 구조는 아래 네 단계입니다.

- 계약금
- 중도금 1
- 중도금 2
- 잔금

각 행에서 관리하는 값은 아래와 같습니다.

- 지급 구분
- 금액(만원)
- 상태
- 지급일
- 지급 경로 / 메모

### 상태 규칙

- `지급완료`: 이미 송금 또는 결제가 끝난 상태
- `예정`: 아직 지급하지 않은 상태

### 현재 반영된 실제 이력

- 계약금
  - 금액: `300만원`
  - 지급일: `2026-04-25`
  - 지급 경로: `국민 632902-01-074940 이하종`
  - 상태: `지급완료`

중도금 1, 중도금 2, 잔금은 현재 예정 상태로 남겨두고 추후 지급 시 갱신합니다.

## 5. 비용 부담 요약 표

예산표 중간에는 현재 시점의 실제 부담 구조를 보여주는 요약 표가 있습니다.

표 항목은 아래 기준으로 계산됩니다.

- `공사비 지급완료`
  - 공사비 지급 현황 표에서 상태가 `지급완료`인 항목 합계
- `공사비 미지급 예정`
  - 공사비 지급 현황 표에서 상태가 `예정`인 항목 합계
- `기구매 합계`
  - 구매 필요항목 표에서 상태가 `구매완료`인 항목 합계
- `미구매 예정 합계`
  - 구매 필요항목 표에서 상태가 `구매예정`인 항목 합계
- `총 예상 필요금`
  - 전체 기준 부담 총액

이 표는 실제로 이미 나간 돈과 앞으로 나갈 돈을 분리해서 보기 좋게 만드는 용도입니다.

## 6. 구매 필요항목 표

하단 상세표는 입주 전후에 필요한 물품을 하나씩 채워 넣는 핵심 영역입니다.

### 컬럼 의미

- `분류`
  - 가전 / 가구 / 수납 / 생활 / 관리 등
- `항목`
  - 실제 구매 대상 이름
- `메모`
  - 브랜드, 모델, 사이즈, 수량 같은 보조 설명
- `예상(만원)`
  - 구매 전 잡아둔 예상 예산
- `실결제(원)`
  - 실제 결제 완료 금액
- `상태`
  - 구매완료 / 구매예정 / 보류
- `구매일`
  - 실제 결제 또는 주문한 날짜
- `결제 경로`
  - 쿠팡, 카드, 계좌이체, 매장 결제 같은 결제 수단 또는 경로

### 상태 규칙

- `구매완료`
  - 이미 결제함
  - 합계 계산 시 `실결제(원)`을 우선 사용
  - 실결제 금액이 비어 있으면 `예상(만원)`을 원화로 환산해 사용
- `구매예정`
  - 아직 구매 전이지만 예산 반영 필요
  - 합계 계산 시 `예상(만원)`만 사용
- `보류`
  - 필요 여부를 아직 판단하지 않았거나 나중에 보기로 한 항목
  - 금액이 0원이면 요약 합계에 포함되지 않음

### 금액 단위 규칙

- `예상(만원)`은 만원 단위 입력
  - 예: `165` 입력 시 `1,650,000원`
- `실결제(원)`은 실제 결제 금액 그대로 원 단위 입력
  - 예: `234740`

이 두 단위를 혼동하면 총액이 크게 틀어질 수 있으니 반드시 구분합니다.

## 8. 0원 보류 항목 관리 방식

문서에는 당장 사지 않더라도 빠뜨리기 쉬운 항목들이 `0원` 상태로 들어 있을 수 있습니다.

예를 들면 아래와 같습니다.

- 전자레인지 / 오븐 / 에어프라이어
- 공기청정기 / 제습기
- 정수기 / 생수 / 커피머신
- 책상 / 의자
- 서랍장 / 수납장
- 방충망 / 미세방충망 보수
- 인터넷 / TV 이전 설치비
- 가전 이전 설치 / 앵글 / 배관
- 욕실 / 세탁 / 청소용품
- 멀티탭 / 공유기 / 소형공구
- 쓰레기통 / 분리수거함
- 옷걸이 / 행거 / 리빙박스
- 밥솥 / 토스터 / 전기포트
- 폐기물 스티커 / 대형폐기물
- 예비비

이 항목들은 필요해지면 금액과 상태를 바꿔서 활성화합니다.

## 9. 인쇄 동작 규칙

문서에는 인쇄 버튼이 들어 있습니다.

현재 인쇄 동작은 아래 순서로 설계되어 있습니다.

1. 인쇄 버튼 클릭
2. `구매 필요항목` 상세표 영역을 숨김 처리
3. 브라우저 인쇄 창 호출
4. 인쇄가 끝나면 화면 상태 복원

즉, 화면에서는 상세표를 볼 수 있지만, PDF나 종이로 출력할 때는 너무 길어지는 하단 상세 영역이 자동으로 빠집니다.

## 10. A4 PDF 출력 기준

이 문서는 A4 기준 PDF 출력에 맞춰 스타일이 조정되어 있습니다.

출력 시 권장 설정은 아래와 같습니다.

- 용지: `A4`
- 방향: `세로`
- 여백: `기본` 또는 `좁게`
- 배경 그래픽: 필요 시 켜기

인쇄 결과에 포함되는 핵심 영역은 아래와 같습니다.

- 상단 제목
- 요약 카드
- 금액 입력된 구매 목록 또는 요약 영역
- 공사비 지급 현황
- 비용 부담 요약

길이가 긴 상세 입력표는 인쇄에서 숨겨집니다.

## 11. 수정 원칙

문서를 수정할 때는 아래 순서를 권장합니다.

1. 공사비 관련 변경이 있으면 먼저 `공사비 지급 현황` 표를 갱신
2. 구매한 물건이 생기면 해당 행의 상태를 `구매완료`로 변경
3. 실결제 금액, 구매일, 결제 경로를 함께 입력
4. 구매 예정만 잡아둘 항목은 `구매예정` 상태 유지
5. 아직 판단하지 않은 항목은 `보류` 또는 0원 유지

## 12. 수정 후 검산 체크리스트

수정 후에는 아래 항목을 꼭 확인합니다.

- 상단 `확정 공사비`가 공사비 지급표 총액과 일치하는지
- `기구매 합계`가 실제 완료 항목만 합산하는지
- `미구매 예정 합계`에 이미 산 항목이 들어가지 않았는지
- `총 예상 필요금`이 공사비 + 기구매 + 미구매 예정으로 맞는지
- 실결제 금액은 원 단위, 예상 금액은 만원 단위로 들어갔는지
- 인쇄 버튼을 눌렀을 때 상세표가 숨겨지는지
- 한글이 깨지지 않는지

## 14. 앞으로의 사용 방식

이 문서는 이제 세종인테리어 기준 최종 예산 관리 문서입니다.

앞으로는 아래 흐름으로 계속 업데이트하면 됩니다.

- 공사비 지급 시 지급표 업데이트
- 물건 구매 시 상태를 `구매완료`로 전환
- 실제 결제 금액과 경로 기록

이 원칙만 지키면 문서가 단순 견적표가 아니라 실제 생활비와 입주비까지 관리하는 운영표 역할을 하게 됩니다.
